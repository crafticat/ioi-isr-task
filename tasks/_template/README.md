# New-task template

Copy this directory to `tasks/<your-task-name>/` and author from there (or just
tell Claude what task you want — it uses this layout via the `iloi-cms-task`
skill).

Expected package layout (canonical import schema — what the CMS importer and the
`verify` workflow consume):

```
tasks/<name>/
├── task.yaml          # start from the one in this template; every model
│                      # solution MUST declare subtask_expected_scores
├── tests.zip          # input_*.txt / output_*.txt (generated by gen/)
├── statements/        # statement PDF(s)
├── solutions/         # model + probe solutions (each listed in task.yaml)
├── generators/        # test generator + GEN plan
├── validators/        # per-subtask input validators
├── managers/          # checker / grader, if the task has one
└── attachments/       # contestant-facing attachments (optional)
```

Ground rules (hard-won — the verify workflow enforces the first):
- **No `subtask_expected_scores`, no verification.** A task with no declared
  expectations fails CI loudly rather than passing vacuously.
- Checker messages must be `translate:success` / `translate:wrong` /
  `translate:partial` tokens, not free English.
- Include probe solutions (per-subtask single-solvers, a 15%-floor probe) so
  every subtask is pinned to the point.
